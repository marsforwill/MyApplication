# 塔罗牌图片修复说明

## 问题诊断

经过详细检查,发现以下四张塔罗牌的图片文件已经损坏:
- **card_10.png** - 命运之轮
- **card_11.png** - 正义  
- **card_12.png** - 倒吊人
- **card_13.png** - 死神

## 损坏原因

这些PNG文件的二进制数据在某个时候被错误地当作UTF-8文本处理,导致:
1. 文件中大量字节被替换为UTF-8替换字符(`EF BF BD`)
2. PNG块的CRC校验和损坏
3. 图片无法被正常的图片查看器和渲染引擎加载

检查结果显示:
- 正常的PNG文件头应该是: `89 50 4E 47 0D 0A 1A 0A`
- 损坏的文件中出现大量: `EF BF BD` (UTF-8替换字符)
- Git历史中的文件也已损坏,说明问题在文件首次提交时就存在

## 临时解决方案

已使用 `card_09.png` (隐者牌)作为临时占位符替换这四张损坏的图片,使应用可以正常运行而不会出现加载错误。

## 永久解决方案

需要获取这四张塔罗牌的正确PNG图片文件:

### 方式一:从原始来源获取
如果您有这些图片的原始来源,请重新下载或导出这些图片。

### 方式二:使用现有的类似尺寸图片
参考其他正常的塔罗牌图片(如card_09.png):
- **尺寸**: 1086 x 1810 像素
- **格式**: PNG
- **文件大小**: 约2-4MB

### 方式三:重新设计或寻找替代图片
在以下网站寻找合适的塔罗牌图片:
- Unsplash
- Pexels  
- 专门的塔罗牌图片资源站

## 替换步骤

1. 准备好新的图片文件(PNG格式,建议尺寸1086x1810)
2. 重命名为对应的文件名:
   - 命运之轮 → `card_10.png`
   - 正义 → `card_11.png`
   - 倒吊人 → `card_12.png`
   - 死神 → `card_13.png`
3. 复制到目录: `entry/src/main/resources/base/media/`
4. 重新构建应用

## 预防措施

为避免类似问题,建议:
1. 在`.gitattributes`中明确标记图片文件为二进制:
   ```
   *.png binary
   *.jpg binary
   *.jpeg binary
   ```
2. 使用Git LFS管理大型二进制文件
3. 确保在传输和复制文件时使用二进制模式
